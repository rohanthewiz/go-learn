## The runner architecture (new files)

```
engine/js-run.js     UMD execute core, shared browser ⟷ CI: strict-mode
                     new Function() exec with captured console + VIRTUAL
                     timers; drains microtasks/timers deterministically
engine/worker-js.js  worker host: importScripts js-run.js, immediate
                     'ready' (nothing to download), promise-queue serial
                     runs, onunhandledrejection swallowed by design
engine/runner-js.js  runner id 'js' — runner-ts.js's watchdog/respawn
                     pattern, lazily spawned on first run()
```

What replaces ts-pure's "the type checker is the harness" gate is
**determinism engineering** (the whole design lives in js-run.js):

- **Virtual timers**: setTimeout/setInterval/clearTimeout/clearInterval
  are shims passed as Function params (bare names in user code resolve to
  them; the host's real timers are unreachable by accident). Callbacks
  fire after the sync body in (due-time, insertion) order; `vnow` jumps to
  each timer's due time — delays are ORDERING, not waiting, so
  `setTimeout(f, 5000)` is instant. Between every fire the drain loop
  yields one REAL macrotask — `setImmediate` (Node) / `MessageChannel`
  (worker), NOT real setTimeout, which browsers clamp to ~4ms when nested
  — so the entire microtask queue (promise chains, awaits) flushes between
  timers, exactly the real event-loop ordering. Intervals reschedule
  BEFORE their callback runs (so self-clearInterval works); >1000 fires
  returns a diagnostic instead of a watchdog kill (or a CI hang — verify
  has no watchdog).
- **Parse vs run are separate try blocks**: a program that fails to parse
  is a "syntax error", but a SyntaxError thrown WHILE running (JSON.parse)
  stays a "runtime error" — the error class can't tell them apart, the
  phase can.
- **Line mapping**: V8 stacks put user line 1 at `<anonymous>:4` (function
  anonymous(params\n) {\n"use strict";\n…), so map = stackLine − 3,
  guarded to [1, srcLineCount]. Non-V8 engines degrade to no position.
- **fmt shim**: lockstep with ts-run.js's PLUS a `typeof v === 'number'`
  branch — JSON.stringify renders NaN/Infinity as "null", unacceptable
  when a coercion lesson must print `NaN`. (Caveat inherited: NaN/undefined
  NESTED in an object still JSON-mangle; lessons avoid checking those.)
- Strict mode + function scope: reruns never leak bindings; `x = 1`
  without declaration is a ReferenceError; NO top-level await (function
  body, not a module) — lessons end with `main();`.
- Unhandled rejections are swallowed (worker `onunhandledrejection`
  preventDefault; verify installs a process-level no-op) — a starter that
  drops a rejection fails its stdout check, the honest symptom, instead of
  killing the worker/CI.

## The track (tracks/js-pure/)

Registration global `GoLearnJSP.lesson` sets `kind:'lesson', lang:'js'`.
All 22 items are lessons (run the file, check stdout); one uses
`starterError: true` — for JS that means the starter fails at RUNTIME
(vs ts-pure's compile-time).

| Category | Items |
|---|---|
| Foundations | hello-javascript, types-coercion, control-flow, functions |
| Collections & Data | arrays-methods, strings-regex, objects, destructuring-spread, maps-sets |
| Functions in Depth | closures, higher-order-functions, this-binding |
| Prototypes & Classes | prototypes, classes, inheritance |
| Iterators & Generators | iterators, generators |
| Async JavaScript | promises, async-await, event-loop |
| Advanced | error-handling*, proxy-reflect |

(* = starterError: the starter's uncaught ValidationError red pane — with
mapped line — is the lesson's opening beat.)

Design principle: every check() pins output only reachable through the
lesson's feature. Standouts: objects pins `retries: 0` (`??` vs `||` made
falsifiable); closures ships the `for (var …) + setTimeout` bug printing
`3 3 3` (virtual timers make it demonstrable) and asserts its absence;
maps-sets counts `"constructor"` honestly through prototype pollution and
pins the fmt shim's exact `Map(3) { … }` rendering; classes proves
`#private` via `JSON.stringify` snapshot; generators pins `pulled: 8` (a
counter inside an infinite generator proving laziness); async-await shows
concurrency by log interleaving (all `start` before first `done`);
event-loop pins the EXACT 5-line total order across sync → microtask →
timer(0) → timer(10); this-binding starters demonstrate lost-`this`
failures inside try/catch so they still run clean. SVG diagrams use
namespaced marker ids `dgArrowJS??` (collision-checked site-wide).
