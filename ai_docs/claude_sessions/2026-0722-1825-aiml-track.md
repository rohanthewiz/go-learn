## What this session did

Added the largest track yet: **`aiml` — "AI & Machine Learning"**, 27
problems on the existing `go-wasm` runner, basics → the current stack.
Zero engine changes. Premise: every other course hands you a library and
`fit()` hides the part that makes someone a data scientist — here the
learner IMPLEMENTS every algorithm in pure Go against a deterministic
harness. Per user directive: covers everything from basics to transformers
(a working one, built in Go), RAG, and RL; deployment deliberately out of
scope (k8s/networking tracks own it). Determinism is a design rule: all
randomness replaced by pinned explicit LCGs or pinned data, so every test
has exactly one right answer.

## Findings worth remembering (all added to the memory gotchas file)

1. **verify.mjs deadlocks under the Bash-tool sandbox**: execFileSync's
   stdin pipe to the native Go runner never delivers — static checks
   print, then it hangs at ~0 CPU. Run verify sandbox-disabled; two
   agents' verify processes stalled this way mid-session (the neural
   agent diagnosed it).
2. **Assign SVG id suffixes centrally in agent prompts** — slug-initial
   schemes collide across parallel agents (several items also mint a
   second warn-colored marker: `dgArrowAIQLw` etc. — fine, still
   namespaced).
3. Placeholder-first parallel authoring survived its biggest run (9
   concurrent agents): one agent saw transient MODULE_NOT_FOUND from a
   neighbor's mid-write file during scoped verify — retry sufficed;
   final full-suite gate run by the main session is the authority.
4. Sound agent deviations worth keeping as patterns: unsorted-order CV
   dataset (lesson clarity), the Q-learning snack terminal (making the γ
   flip real required a second reward), bit-identical vectors for
   float-safe tie-break tests, testing the LCG stream itself as case 1.
